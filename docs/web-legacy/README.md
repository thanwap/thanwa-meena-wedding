the new web project from scratch. Assets are preserved under `./assets/`.

  `Geist` (`--font-geist`)

## 5. Authentication — Google OAuth via Auth.js v5

`web/auth.ts`:

```ts
import NextAuth from 'next-auth'
import Google from 'next-auth/providers/google'

declare module 'next-auth' {
  interface Session { idToken?: string }
}
declare module '@auth/core/jwt' {
  interface JWT { idToken?: string }
}

export const { handlers, auth, signIn, signOut } = NextAuth({
  providers: [Google],
  callbacks: {
    jwt({ token, account }) {
      if (account?.id_token) token.idToken = account.id_token
      return token
    },
    session({ session, token }) {
      session.idToken = token.idToken
      return session
    },
  },
})
```

Key behaviour: the **Google `id_token`** is captured on first sign-in and
attached to `session.idToken`, so server actions can forward it as a Bearer
token to the .NET API (which validates it as a Google JWT).

Route handler — `web/app/api/auth/[...nextauth]/route.ts`:

```ts
import { handlers } from '@/auth'
export const { GET, POST } = handlers
```

### Proxy / route protection

Next.js 16 uses `proxy.ts` (not `middleware.ts`). `web/proxy.ts`:

```ts
import { auth } from '@/auth'
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export async function proxy(request: NextRequest) {
  const session = await auth()
  if (!session) {
    const callbackUrl = encodeURIComponent(request.nextUrl.pathname + request.nextUrl.search)
    return NextResponse.redirect(new URL(`/api/auth/signin?callbackUrl=${callbackUrl}`, request.url))
  }
  return NextResponse.next()
}

export const config = {
  matcher: ['/admin/:path*'],
}
```

Google Cloud Console setup:
- OAuth 2.0 Web client
- Authorized redirect URI: `https://frommeenatothanwaforever.com/api/auth/callback/google`
  (and `http://localhost:3000/api/auth/callback/google` for dev)

## 6. Calling the .NET API

Pattern: **server actions** in `app/admin/configs/actions.ts`. Each action
pulls the Google ID token from the session and forwards it as a Bearer token,
with `cache: 'no-store'` for reads.

```ts
'use server'
import { auth } from '@/auth'

async function getIdToken(): Promise<string> {
  const session = await auth()
  if (!session?.idToken) throw new Error('Not authenticated')
  return session.idToken
}

const API = process.env.DOTNET_API_URL!

export async function getConfigs() {
  const token = await getIdToken()
  const res = await fetch(`${API}/api/configs`, {
    headers: { Authorization: `Bearer ${token}` },
    cache: 'no-store',
  })
  if (!res.ok) throw new Error(`Failed to fetch configs: ${res.status}`)
  return res.json()
}
```

CRUD endpoints used by the admin page:
- `GET    {DOTNET_API_URL}/api/configs`
- `POST   {DOTNET_API_URL}/api/configs`            body `{ key, value, type }`
- `PUT    {DOTNET_API_URL}/api/configs/{id}`       body `{ key, value, type }`
- `DELETE {DOTNET_API_URL}/api/configs/{id}`

All require `Authorization: Bearer <google id_token>`.

Admin pages live at `/admin` and `/admin/configs` (protected by `proxy.ts`),
with client components `ConfigsClient`, `ConfigForm`, `ConfigTable` and a
`types.ts` exporting `ConfigDto`.

## 7. CI/CD pipeline

Two workflows under `.github/workflows/`. The web one is the relevant model.

### `deploy-web.yml` — triggered on **semver tag push** (`X.Y.Z`)

```yaml
on:
  push:
    tags: ['[0-9]+.[0-9]+.[0-9]+']

concurrency:
  group: deploy-web-${{ github.ref }}
  cancel-in-progress: true

jobs:
  test-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: web/package-lock.json
      - run: npm ci          # working-directory: web
      - run: npm run lint    # working-directory: web
      - run: npm test -- --ci # working-directory: web
      - run: npm install -g vercel@50.39.0
      - run: vercel pull --yes --environment=production --token=$VERCEL_TOKEN
      - run: vercel build --prod --token=$VERCEL_TOKEN
      - run: vercel deploy --prebuilt --prod --token=$VERCEL_TOKEN
```

GitHub Action secrets required: `VERCEL_TOKEN`, `VERCEL_ORG_ID`,
`VERCEL_PROJECT_ID`. Auto-deploys from git push are disabled via
`web/vercel.json` `ignoreCommand: "exit 1"`, so deploys only happen via this
workflow on tag.

`deploy-api.yml` is the sibling pipeline that builds the .NET 10 Docker image
and deploys to Render — kept untouched.
